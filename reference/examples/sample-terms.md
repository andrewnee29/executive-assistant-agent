### Commentary for the builder:

- **Projects section** captures institutional knowledge that's rarely written down
- **Tool descriptions** include the "note" about transcription ambiguity — this is what makes corrections work
- **Acronym definitions** include context ("at this company") because the same acronym can mean different things
- **Corrections table** is the most operationally valuable part — built up from actual transcription errors encountered
- Entries are added when they first appear in meetings and refined as more context accumulates
